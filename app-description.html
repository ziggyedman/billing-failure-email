Intro ~20s
One-line pitch: A self-contained interactive tutorial that walks a developer through sending a real billing-failure email — live template editing, live preview, real send.

The prompt was to build something that teaches developers how to send a billing-failure email using React Email and Resend.

Instead of writing a static doc, I built an interactive wizard — a four-step tutorial app where you actually run every piece of the implementation.

The four steps ~30s

The steps mirror the real production path:

1 Get a Resend API key. 2 Verify a sending domain. 3 Edit the email template and preview it live. 4 Set the customer details and fire a real send.

Each step is self-contained. Jump to any step from the sidebar at any time. Progress and inputs persist in localStorage across refreshes.

Step 3 — template editing ~40s

Step 3 is where most of the interesting work happens.

The left column has two tabs. Try it is a full template editor — the actual emails/billing-failure.tsx file, editable right in the browser. The right column is a React Email component library — all 14 components, each with a description, a when-to-use note, and a copyable snippet.

You copy a snippet, paste it into the editor, click Render preview, and see the rendered email immediately. Hit Reset to original to discard all changes. Whatever you render here is what gets actually sent in Step 4.

The How it's built tab walks through the template file section by section — file structure, props interface, layout, branding, status banner, CTA, footer, and why every style is an inline object. Written as a general reference for engineers building their own email templates, not just this one.

Step 4 — send ~30s

Step 4 separates customer data from template structure — intentionally.

You edit the customer details — name, amount, card last 4, failure reason — and see the resend.emails.send() call update live.

Right below it, a live email preview shows exactly what will be sent. If you rendered a custom template in Step 3, it loads that version directly, with a Custom badge in the header. Otherwise it shows the default template.

Then you enter a recipient and click send. The response comes back with a Resend message ID.

Wrap up ~15s

The repo is the tutorial. Fork it, drop the wizard UI, keep emails/billing-failure.tsx and the send route, wire it to a Stripe webhook — it's production-ready as-is.
