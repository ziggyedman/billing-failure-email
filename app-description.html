Intro ~20s
What this is: an interactive tutorial that walks a developer through sending a real billing-failure email. Live template editing, live preview, real send.
  • Built with Next.js, React Email, and Resend
  • Not a static doc. Every step runs real code from the repo
  • Four steps. Mirror the exact path you'd follow in a real product
The four steps ~25s
  • Step 1 — Create a Resend account, generate an API key, paste it in
  • Step 2 — Verify your sending domain with DNS records
  • Step 3 — Edit the email template and preview it live
  • Step 4 — Set customer details, preview the email, fire the send
  • Jump to any step at any time. Progress saves in localStorage
Step 3 — Template ~35s
  • Left column: Try it tab — the actual emails/billing-failure.tsx file, editable in the browser
  • Right column: component library — 14 React Email components, each with a description, when-to-use, and a copyable snippet
  • Copy a snippet, paste it in, click Render preview — see the result instantly
  • Whatever you render here is what gets sent in Step 4
  • How it's built tab — section-by-section walkthrough of the template file, written as a general reference for building your own
  • Props interface, layout, branding, CTA, footer, inline styles — and why each decision matters
Step 4 — Send ~30s
  • Edit customer details — name, amount, card, failure reason, retry date
  • These come from your payment provider in a real integration. A Stripe invoice.payment_failed event, for example
  • Click Apply — the code snippet updates, the preview re-renders with the new values
  • Live email preview shows exactly what will be sent. Custom badge if you edited the template in Step 3
  • Enter a recipient, click Send — response comes back with a Resend message ID
Wrap up ~15s
  • The repo is the tutorial. Fork it, drop the wizard UI
  • Keep emails/billing-failure.tsx and the send route, wire to a Stripe webhook — production-ready
  • Happy to go deeper on any part pause
